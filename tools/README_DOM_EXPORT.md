# Browser DOM Export Scripts

Simple JavaScript snippets to export the **current rendered DOM** from a web page directly from your browser.

These scripts allow you to capture the HTML that the browser sees after JavaScript execution, which is useful for:

- Web scraping experiments
- DOM inspection
- Creating offline copies of pages
- Building browser automation tools
- Extracting data from dynamic websites

No external libraries or AI tools are required.

# Usage

1. Open any webpage.
2. Open Developer Tools:
   - Chrome / Firefox: `F12` or `Cmd + Option + I` (Mac)
   - Safari: Enable Developer Menu first
3. Go to the **Console** tab.
4. Paste one of the scripts below.
5. Press Enter.
6. The browser will download the rendered page DOM as `page.html`.

---

# Chrome + Firefox + Safari

Use this version for maximum browser compatibility.

```javascript
const html = document.documentElement.outerHTML;

const blob = new Blob([html], { type: "text/html" });

const url = URL.createObjectURL(blob);

const a = document.createElement("a");
a.href = url;
a.download = "page.html";

document.body.appendChild(a);
a.click();

document.body.removeChild(a);

URL.revokeObjectURL(url);
```

---

# Chrome Only

A shorter version that works reliably in Chrome.

```javascript
const html = document.documentElement.outerHTML;

const blob = new Blob([html], { type: "text/html" });

const a = document.createElement("a");
a.href = URL.createObjectURL(blob);
a.download = "page.html";

a.click();
```

# What Does This Export?

This exports:

✅ The current DOM  
✅ Content generated by JavaScript frameworks  
✅ Dynamically loaded elements  
✅ Visible page structure  
✅ HTML after the browser finishes rendering

Example:

A React website:

```
Original Server HTML
        |
        v
JavaScript Application Loads
        |
        v
Browser Builds DOM
        |
        v
document.documentElement.outerHTML
        |
        v
Exported page.html
```

---

# Important Notes

## This is NOT the original source HTML

`outerHTML` captures the **live browser DOM**, not the raw response from the server.

For modern websites this is usually preferred because it includes:

- Loaded data
- Rendered components
- Client-side generated content
